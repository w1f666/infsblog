CSS 最容易失控的时刻，往往不是项目刚开始，而是页面和组件数量逐步增加之后。这个阶段的问题不是样式写不出来，而是谁覆盖谁已经说不清了。

我现在更倾向于把样式分成几个层：基础变量、通用布局、组件样式、状态修饰。先把每一层的职责划开，后面需要改动时就不会总想着提高选择器优先级去“压过去”。

另一个很重要的习惯是避免把组件样式写成依赖页面结构的形式。只要一个按钮必须放在某个父级下面才能正常显示，这个组件通常就已经开始变脆了。

样式系统越成熟，越应该让人能预测变化的影响范围。一个类名改了哪里，一个变量影响哪些组件，最好都能大致说清。

前端的很多维护成本，其实都藏在这种看起来很小的覆盖细节里。
